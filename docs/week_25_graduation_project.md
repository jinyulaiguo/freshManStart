## Day 170：为研究助手配备全套 Harness 三层防护隔离环
*   **核心知识点**：
    *   **Harness 三层合围**：
        1. **执行前（Guides）**：挂载 `research_spec.yaml`，限定最大爬取站点数与调用资费；
        2. **执行中（Constraints）**：在 Docker 容器隔离沙箱内运行网页数据爬取与数据分析脚本，降权运行；网络层实行严格白名单访问；
        3. **执行后（Sensors）**：在写手 Agent 生成报告后，审计 Agent 强制审查报告中的事实引用脚注是否 100% 能在 RAG 库中找到逻辑蕴含，缺失则打回重写。
*   **Agent 核心关联**：向面试官证明你的系统不是一个实验室玩具，而是一个“控 AI 不失控”的具备高安全性、可直接部署的商业化级生产系统。
*   **🎯 过关验证标准**：故意在爬取回来的数据中注入恶意的 Python 删盘脚本，验证沙箱容器物理阻断了恶意脚本，并在审计 Agent 的引用校验中判定由于无合理数据源支撑直接报错自愈重新运行。

## Day 172：进行 100 组 Golden Dataset 自动化回归测试与生成评测报告
*   **核心知识点**：
    *   **大规模自动化回归测试**：运行自动化评测脚本 `run_eval.py`，并发加载 100 个由真实问题组成的测试集。
    *   **自动化评估指标汇总**：统计分析回答忠实度、检索召回率、工具 $F_1	ext{-score}$、以及单次任务平均美元开销。
    *   **最终生成高质量的可视化 PDF/Markdown 评测白皮书**。
*   **Agent 核心关联**：这是你向面试官展示的客观技术肌肉。以详实的数据指标（如“本系统在 100 组测试集上达到了 $F_1 = 0.92$，回答忠实度 96%，平均单次耗时 12 秒，运行成本降低 35%”）证明你开发出的 Agent 的卓越品质。
*   **🎯 过关验证标准**：并发运行 100 个测试用例，收集全部指标数据，自动生成一份包含详尽图表、回归变化比对的 Markdown 评测白皮书文件 `eval_whitepaper.md` 并落盘。
