📖 什么是 Tool Runner？

在 AI Agent 架构中，大模型（LLM）不能直接运行代码或访问数据库。大模型必须通过 Function Calling 输出 JSON 参数来声明调用外部工具的意图。


Tool Runner（工具调度器） 是本地系统的中央执行网关。它作为大模型与本地核心函数之间的接口，负责将大模型的文本意图转化为本地代码的运行结果，并保障整个运行过程的类型安全与可观测性。

🎯 Day 14 终极实战交付标准
模块 1
工具定义与安全注册
使用 Pydantic 声明参数，通过装饰器注册函数，利用 model_json_schema() 导出 JSON Schema。
模块 2
入参拦截与安全校验
解析并校验大模型返回的 JSON 参数，拦截类型不合规或多字段逻辑冲突的输入。
模块 3
异步并发引擎
调度流程和工具全面 async 化，遇到多工具调用时用 asyncio.gather 并发非阻塞执行。
模块 4
可观测监控与异常处理
暴露生命周期 Hook 回调，遇到异常用 raise ... from 包装并利用 traceback 记录到日志。

💡 对应知识点

Pydantic 基础定义、字段别名与 Field 配置，JSON 反序列化。

❓ 解决了什么痛点

大模型返回的 JSON 参数是未经验证的文本字符串，若直接传入本地函数执行，极易因类型不匹配或参数缺失而发生运行时崩溃。

🔬 核心逻辑本质

外部边界输入转换：将非结构化的大模型预测文本，转换为本地语言环境支持的、具有强类型约束的入参对象。
