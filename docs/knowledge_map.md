# 🗺️ AI Agent 驾驭工程 · 26 周知识地图

> 本地图用于解决一个核心问题：**"我这周学的东西到底有什么用？"**
>
> 每周开始前看一眼，心中有锚，学习不慌。

---

## 阶段一：Python & LLM 地基（W1 – W5）

> 🎯 阶段使命：让你从"看不懂框架源码"变成"能徒手写出一个最小 Agent"。

| 周 | 学什么 | 解决什么痛点 | 后续哪里要用到 |
|---|---|---|---|
| W1 | 字典深度操作、装饰器、OOP 魔法方法、`Protocol`、动态反射 `getattr` | Agent 配置全是嵌套字典，框架全是装饰器和 `__call__`，不学这些就是文盲读源码 | W5 手写 ReAct 时要用字典映射工具；W10 LangGraph 的 `@tool` 就是装饰器 |
| W2 | TypedDict、Pydantic 校验与 JSON Schema 导出、async/await 并发、回调钩子 | LLM 吐出的参数不校验就炸；3 个工具串行调用延迟 ×3；出了 Bug 没钩子就是黑盒 | W5 Function Calling 的参数声明；W10 LangGraph State 定义；W16 链路追踪的底层机制 |
| W3 | Transformer 直觉理解、Tokenizer 实操、幻觉成因、API 调用 | 面试必问"为什么会幻觉"；不懂 Token 就算不清成本 | W14 Token 成本优化的理论基础；W16 监控指标的设计依据 |
| W4 | CoT 思维链、Few-shot、结构化输出、正则提取 JSON | LLM 输出格式不可控，混着自然语言的 JSON 直接 `json.loads()` 必崩 | W5 解析 ReAct 的 Thought/Action/Observation；W7 RAG 生成答案的格式约束 |
| W5 | **手写 ReAct 循环**、Function Calling 规范、并行工具调用 | 不手写一遍就永远不知道 Agent 底层在做什么，面试画不出来流程图 | W10 用 LangGraph 重写时能理解"框架帮你省了哪些事" |

> ✅ **里程碑**：一个不依赖任何框架、能调用 3 个工具的最小可运行 Agent。

## 阶段二：知识与记忆系统（W6 – W9）

> 🎯 阶段使命：让 Agent 从"什么都不知道的白痴"变成"能读文档、记住历史的助手"。

| 周 | 学什么 | 解决什么痛点 | 后续哪里要用到 |
|---|---|---|---|
| W6 | Embedding 原理、向量数据库（Chroma/Milvus）、相似度检索 | 没有向量化就无法做语义搜索，只能做关键词匹配，召回率极低 | W7-W8 所有 RAG 的底层存储；W9 长期记忆的向量召回 |
| W7 | RAG 分块策略（固定/递归/语义）、Retrieve-Augment-Generate 全流程 | Agent 读不了本地 PDF/文档，只能靠大模型自己的训练数据瞎编 | W12 规划型 Agent 执行子任务时需要检索背景知识 |
| W8 | Rerank 重排序、HyDE、GraphRAG 知识图谱、Tool Retrieval | 简单 RAG 召回一堆噪声文档；跨文档的关联推理完全做不了 | W13 工具太多时的动态检索；W19 多 Agent 各自检索不同知识源 |
| W9 | 短期记忆（滑动窗口）、长期记忆（向量摘要）、跨会话持久化 | 聊了 10 轮就忘了前面说的话；换个会话就不认识你了 | W11 HITL 断点恢复需要记忆；W14 上下文压缩依赖记忆摘要 |

> ✅ **里程碑**：一个能读文档、能记住历史对话的知识型 Agent。
>
> 🔵 **贯穿项目启动**："AI 研究助手"从本阶段开始构建，后续每周迭代。

## 阶段三：框架与核心工程（W10 – W15）

> 🎯 阶段使命：引入工业级框架，补齐规划与自纠错能力，建立"改了代码能证明变好了"的评测体系。

| 周 | 学什么 | 解决什么痛点 | 后续哪里要用到 |
|---|---|---|---|
| W10 | LangGraph StateGraph、节点/边/条件分支 | 手写 Agent 逻辑一复杂就变成意大利面条代码，无法维护 | 后续所有阶段的 Agent 都在 LangGraph 上构建 |
| W11 | HITL 人工审批断点、子图、Checkpoint 持久化 | 高风险操作（发邮件/转账）不让人确认就直接执行，出了事谁负责？ | W18 部署到生产环境的安全前置条件；W23 沙箱的审批触发 |
| W12 | Plan-and-Execute、Reflexion 自纠错、LLM-as-Critic | ReAct 遇到复杂任务效率极低（一步一步试）；Agent 犯了错不会自己改 | W19 多 Agent 中"审查者"角色的核心能力；W24 Harness 反馈层的纠错逻辑 |
| W13 | MCP 协议 Server 开发、Tool 设计最佳实践 | 每接一个新工具就要改一堆代码；工具描述写不好 LLM 就调不准 | W20 A2A 跨系统时的标准工具暴露方式；W21 Supervisor 动态分配工具 |
| W14 | 上下文分层、Prompt Cache、大小模型路由 | 一个复杂对话烧掉几十块钱 Token 费；长上下文导致响应延迟 5 秒+ | 直接决定毕业项目能否在企业低成本落地 |
| W15 | Golden Dataset、LLM-as-Judge、CI/CD 评测流水线 | 改了一行 Prompt，不知道 Agent 是变好了还是变烂了 | W24 Harness 反馈层的回归测试基准；面试核心差异化武器 |

> ✅ **里程碑**：具备框架编排 + 规划能力 + 自动化评测的完整 Agent 系统。
>
> 🟢 **开源参与启动**：开始读 LangGraph 源码，遇到问题提 Issue/PR。
>
> 🔴 **W15 结束后开始投简历**：此时你已具备大多数 Agent 岗位的面试资格。

## 阶段四：生产工程化（W16 – W18）

> 🎯 阶段使命：从"我电脑上能跑"变成"服务器上稳定运行、出了问题 10 秒内定位"。

| 周 | 学什么 | 解决什么痛点 | 后续哪里要用到 |
|---|---|---|---|
| W16 | LangSmith/Phoenix Trace 追踪、Grafana 监控看板 | Agent 内部推理了 8 步，第 5 步出错了，不做 Trace 根本不知道哪步出的问题 | W24 Harness 反馈层的运行时监控数据来源 |
| W17 | 指数退避重试、熔断器、限流、Fallback 降级 | 大模型 API 偶尔超时/限流，一个工具挂了整个 Agent 就崩溃 | 保障 W19-W21 多 Agent 并行时的系统稳定性 |
| W18 | FastAPI 接口封装、Docker 容器化、云端部署 | "在我电脑上跑得好好的"——换台机器就炸 | 后续所有多 Agent 系统的独立部署底座；毕业项目的云端展示 |

> ✅ **里程碑**：可容器化部署、有监控、有容灾的生产级 Agent 系统。

## 阶段五：多 Agent 系统与协议（W19 – W21）

> 🎯 阶段使命：从"一个人干所有活"变成"一个团队各司其职"。

| 周 | 学什么 | 解决什么痛点 | 后续哪里要用到 |
|---|---|---|---|
| W19 | 顺序/层级/辩论/投票四种协作模式、Handoff 任务交接 | 单 Agent 上下文塞太多就犯蠢；搜资料、写报告、审校全让一个 Agent 干，质量很差 | 毕业项目"AI 研究助手"拆分为搜索/分析/写作三个子 Agent |
| W20 | A2A 跨系统协议、MCP + RESTful 双协议暴露、并发压测 | 公司 A 的 Agent 和公司 B 的 Agent 没法对话，各自为战 | 企业级数据中台 Agent 的跨部门协作架构 |
| W21 | Supervisor 中央调度、子 Agent 资源隔离、意图防漂移、冲突仲裁 | 多 Agent 跑着跑着就跑偏了，或者两个 Agent 意见冲突时死锁 | 毕业项目的最终架构形态：Supervisor + 多专项 Agent |

> ✅ **里程碑**：多 Agent 协作系统，能分工、能通信、能仲裁。

## 阶段六：Harness Engineering（W22 – W24）

> 🎯 阶段使命：给 Agent 套上"缰绳"——让它强大但不失控。
>
> 核心公式：**Agent = Model + Harness**
>
> 核心哲学：Agent 犯错时，不换更强的模型，而是改进 Harness。

| 周 | 学什么 | 解决什么痛点 | 对应 Harness 层级 |
|---|---|---|---|
| W22 | SPEC 任务定义、Rule 硬性约束、Skill 技能包、AGENTS.md | Agent 执行前没有任何规则限制，想干什么就干什么 | **引导层（Feedforward）**：执行前告诉它边界在哪 |
| W23 | Docker 安全沙箱、文件/网络权限控制、输入输出护栏、Prompt Injection 防护 | Agent 生成的代码直接在宿主机运行，可能 `rm -rf /` 删掉一切 | **约束层（Constraints）**：执行中限制它能做什么 |
| W24 | 审计 Agent 监督、Reflexion 自动纠错、语义缓存、Token 预算熔断 | 报错了没人管就卡死；重复问同一个问题每次都重新调 API 烧钱 | **反馈层（Sensors）**：执行后验证结果并纠错 |

> ✅ **里程碑**：Harness 三层防护完整覆盖的生产级 Agent 系统。

## 🛡️ 抗焦虑锚点

在漫长学习中产生怀疑时，看这里：

| 你的焦虑 | 校准答案 |
|---|---|
| "天天写 Pydantic 和异步，离 Agent 好远" | 你现在写的每一行 Pydantic，都是 W13 标准工具层的地基。不写异步，3 个工具串行调用延迟 ×3。 |
| "有 Dify/Coze 拖拽平台，为什么还要学 LangGraph" | 拖拽做不了"SQL 写错自动重试"和"人工审核断点重跑"。企业高薪招你，是让你在代码级驾驭复杂逻辑。 |
| "评测 Eval 好枯燥，真有用吗" | 90% 的竞争者被问"怎么证明改好了"只会说"我试了几个 case 感觉可以"。你有 50 条测试集 + CI 流水线，这是分水岭。 |
| "沙箱和权限是运维的事吧" | 当 Agent 有写码和执行权限时，失控 = 灾难。掌握 Harness 是 2026 年最稀缺的架构能力。 |
