# 📅 Week 1: Python 基础、函数进阶与面向对象

> **第一周目标**：精通字典与数据结构，彻底掌握装饰器及 OOP 接口设计，能够读懂主流框架的底层对象调用。

---

## Day 1：基础语法与核心数据结构
*   **核心知识点**：
    *   变量与基本数据类型（Str/Int/Float/Bool）
    *   列表（List）、元组（Tuple）、集合（Set）的常用操作
    *   **字典（Dict）深度操作**：嵌套字典提取、`get()` 默认值、字典推导式、处理 `KeyError`。
*   **Agent 核心关联**：Agent 中的 Message 结构、LLM 接口返回的 Payload、系统配置几乎全是嵌套字典。
*   **🎯 过关验证标准**：手写一个函数，输入一个多层嵌套的字典，能够安全地提取指定路径的值（如 `data['meta']['usage']['total_tokens']`），在路径不存在时优雅返回自定义默认值而不报错。

---

## Day 2：控制流、正则提取与 JSON 容错解析（🌟 新增）
*   **核心知识点**：
    *   条件判断（if-elif-else）与循环（for/while）
    *   函数的定义、形参与实参、返回值
    *   **异常处理机制**（`try-except-finally`）与捕获 `JSONDecodeError`
    *   **正则表达式基础（`re` 模块）**：使用 `re.search` / `re.findall` 提取特定文本（如 Markdown ` ```json ... ``` ` 块）
    *   **非标准 JSON 的容错清洗**：清洗换行符、处理垃圾字符、防范大模型输出解析失败。
*   **Agent 核心关联**：LLM 输出的往往是带有自然语言的混合文本，必须使用正则提取出结构化的 JSON 部分。如果 JSON 稍有损坏，异常处理能保证 Agent 不直接崩溃。
*   **🎯 过关验证标准**：写一个函数，输入一段大模型混合输出的文本（如 `"这是结果：\n\n\`\`\`json\n{\"action\": \"calc\", \"args\": 42}\n\`\`\`\n请查收。"`），使用正则表达式提取该 JSON 字符串，并通过 `try-except` 捕获可能的 JSON 解析异常，安全地解析为 Python 字典。

---

## Day 3：函数进阶与装饰器（P1 优先级）
*   **核心知识点**：
    *   `*args` 与 `**kwargs` 的参数接收与解包
    *   闭包（Closure）的概念与内存机制
    *   **自定义装饰器**：无参装饰器、带参数的装饰器
    *   函数作为一等公民（传参、返回、存储在字典中）
*   **Agent 核心关联**：`*args/**kwargs` 用于实现通用的 Tool 调用分发；`@tool` 装饰器是所有框架注册工具的标配。
*   **🎯 过关验证标准**：手写一个带参数的装饰器 `@retry(times=3)`，当被装饰的函数抛出异常时，能够自动重试指定的次数，并打印每次重试的参数。

---

## Day 4：面向对象（OOP）魔法方法（P1 优先级）
*   **核心知识点**：
    *   类与实例、继承与多态、属性与方法的私有化
    *   **核心魔法方法**：`__init__`（初始化）、`__call__`（使实例像函数一样可调用）、`__repr__` / `__str__`（对象可视化表达）
*   **Agent 核心关联**：`__call__` 是各种 Chain/Agent 实例直接调用的秘密；`__repr__` 决定了调试时控制台打印的日志是否清晰。
*   **🎯 过关验证标准**：实现一个名为 `SimpleChain` 的类，初始化时接收一个提示词模板，实现 `__call__` 方法，使得 `chain("输入数据")` 能够直接输出格式化后的提示词。

---

## Day 5：接口抽象、设计模式与动态反射（🌟 新增）
*   **核心知识点**：
    *   抽象基类（`abc.ABC` 与 `@abstractmethod`）
    *   **结构化子类型 `Protocol`**（PEP 544 鸭子类型静态检查）
    *   **动态反射机制**：使用 `getattr(object, name)` 动态获取并调用类方法，使用 `globals()` 或 `locals()` 进行函数字典映射，动态执行工具。
    *   Enum（枚举）的基本用法与限制
    *   常用设计模式：单例模式、工厂模式（动态创建 Agent）、观察者模式。
*   **Agent 核心关联**：Agent 动态反射使得我们能根据大模型输出的工具名称字符串直接定位到对应的工具方法，避免硬编码大量 `if-else`。
*   **🎯 过关验证标准**：创建一个包含多个工具方法（如 `send_email`, `search_web`）的类，写一个动态执行函数，接收工具名称的字符串和参数，使用 `getattr` 动态定位并执行该方法；若方法不存在，安全地返回错误提示。
