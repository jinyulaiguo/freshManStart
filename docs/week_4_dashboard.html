Self-Consistency 投票 (Day 22) CoT 注意力分配机制、Few-shot 隐式计算原理、Temperature 概率采样干预机制 复杂规划或选项路由时，单步 CoT 容易偏离逻辑；且 Temperature=0 导致自一致性投票失效。 高 Temperature 并发采样多条路径推理，在 Python 侧以多数票原则输出确定性结果。
Pydantic 运行时契约 (Day 23) Structured Outputs 词表 Logits 遮罩 (Logits Masking) 技术、静态与动态运行时契约 大模型输出格式不可控，容易发生字段缺失或类型溢出，导致下游程序崩溃。 使用 BaseModel 定义强类型契约引导严格输出，拦截不合规响应并提取结构化 ValidationError。
脏 JSON 容错修复 (Day 24) 正则非贪婪边界寻址、语法解析状态机与栈式自愈算法 非严格模式或端侧模型常输出 Markdown 包裹、缺少尾大括号或包含尾逗号。 正则提取 boundaries，手写 Parsing Stack 模拟压栈寻找最佳闭合点原位修复，节省 API 调用成本。
Jinja2 模板与防注入 (Day 25) 指令逃逸劫持原理、提示词沙盒化与防注入过滤模型 提示词硬编码在 Python 代码中难维护；恶意用户在简历文本中夹带指令逃逸破坏 Agent 决策。 独立 .jinja 文件解耦；设计安全护栏检测敏感词并转义，进行拦截。
HTTPX 并发连接池 (Day 26) TCP 三次握手与 TLS 握手开销、HTTP/1.1 Keep-Alive 复用、网络异常链体系 高频网络请求频繁建立 TCP 套接字导致延迟高；底层网络网络波动导致系统异常中断。 调优 Limits 与 Keep-Alive；封装底层网络报错为自定义 ToolNetworkException。
熔断器与并发速率限制 (Day 27) 熔断状态机模式、限流控频算法（令牌桶 vs 漏桶） 第三方 API 持续崩溃时，Agent 频繁自动重试导致线程卡死并产生巨额无用计费。 设计 @circuit_breaker 状态转移装饰器；配合令牌桶及 Semaphore 强制控频限速。
Week 4 每日知识链路与 AI 理论学习方向
Day 22 CoT、Few-shot 样本库与 Self-Consistency 异步并行投票
核心要点：CoT注意力分配机制与思维链 Token 引导原理；In-Context Learning 上下文学习隐式优化；Temperature 温度参数作用于 Logits Softmax 的采样机制；Self-Consistency 多路径采样的多数票表决数学原理。
🔬 理论突破：深入理解为什么 Temperature 设为 0 会导致自一致性投票直接崩溃失效；如何利用向量库建立动态召回，削减静态 Example 引入的 Prior bias（先验偏好偏置）。
🎯 过关标准：实现 SelfConsistencyEvaluator，以异步并发（asyncio.gather）请求 5 次包含 CoT 推理的 Prompt，提取选项或核心数值，在 Python 中实现高效率多数票选择，并过滤非法响应。
Day 23 大模型原生结构化输出与 Pydantic 边界类型契约
核心要点：JSON Mode与Strict严格模式差异；底层词表 logits 遮罩（Logits Masking）数学原理；Pydantic 运行时类型契约定义、校验与 ValidationError 解析。
🔬 理论突破：掌握 logits masking 的本质——解码器如何通过屏蔽非法词表 token 强制大模型只输出 100% 符合 JSON Schema 的串，理解运行时类型校验如何构成系统边界的安全哨兵。
🎯 过关标准：使用 Pydantic 声明包含嵌套结构和字段校验的 UserInfo。强制大模型输出此结构，故意注入非法参数触发 ValidationError 并解析格式化报错信息。
Day 24 防御性 JSON 解析器与脏 JSON 格式化语法纠错引擎
核心要点：正则非贪婪检索与语法边界提取；基于 Parsing Stack（压栈/出栈）的状态机括号闭合补齐算法；尾逗号、非法单引号以及 Markdown 干扰的本地状态转移原位修复。
🔬 理论突破：理解编译原理中括号匹配压栈出栈对截断数据的恢复原理，掌握本地语法自愈对减少 API 重连延迟、压低系统成本的工程经济价值。
🎯 过关标准：手写 robust_json_parser 解析引擎，能够将故意被破坏的脏 JSON 字符串（包含缺失尾大括号、单引号包裹、包含 Markdown 干扰信息）全部自动修复并成功转换为合法的 Python 字典。
Day 25 生产级 Prompt Jinja2 模版解耦与动态上下文防注入架构
核心要点：Jinja2 模版逻辑控制；提示词物理与逻辑解耦；提示词注入攻击（Prompt Injection）的逃逸机理与边界对抗安全护栏设计。
🔬 理论突破：剖析指令与数据混合架构下的安全缺陷——为什么外部传入的 Observation 能逃逸边界劫持决策流；建立沙盒式输入清洗模型。
🎯 过关标准：设计一个包含工具定义、历史记录及任务的 Jinja2 模板，编写 Python 渲染并过滤用户输入，输入恶意注入指令验证能成功安全拦截。
