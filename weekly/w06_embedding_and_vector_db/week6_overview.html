Week 6 · 宏观总览

Embedding 与向量数据库

精通文本向量化的底层数学表征，掌握 HNSW 索引构建与优化，设计高吞吐并发向量化导入流水线，具备元数据 Pre-Filtering 检索系统的开发能力。

贯穿性项目

AI 研究助手 — 底层知识存储与联合检索服务

本周的核心交付物是为"AI 研究助手"搭建一套完整的知识库向量化存储与相似度联合检索系统。该系统需要完成从"非结构化文本清洗 → 并发批量向量化导入 → HNSW 索引构建 → 元数据联合 Pre-Filtering 检索"的全链路闭环。

最终交付标准：在高并发下，系统能稳定在 15ms 内完成带元数据约束的 Top-K 相似度检索。

Day

文本向量化数学直觉与表征限制

掌握余弦相似度与欧氏距离的底层数学公式与几何物理意义。理解维度裁剪（Matryoshka）技术的存储优化原理。量化分析反讽/转折等语义在高维空间的表征退化现象。

数学基石 Cosine Similarity L2 Distance Matryoshka numpy 离线计算
37
Day

Qdrant 向量数据库原理与本地 HNSW 索引部署

部署 Qdrant 本地 Docker 实例。掌握 HNSW 分层导航小世界索引的多层图检索原理。通过 Python SDK 管理 Collection 生命周期，并发写入 10,000 条向量进行索引构建时间基准测试。

索引引擎 HNSW IVF Docker qdrant-client
38
Day

相似度检索与元数据联合过滤

对比 Post-Filtering 与 Pre-Filtering 两种路径的性能差异。掌握 Qdrant Payload Indexing 机制，实现 category / read_level / created_at 三字段联合过滤。

检索质量 Pre-Filtering Post-Filtering Payload Index
39
Day

分批向量化、批量插入与限流自适应保护

设计并发批量嵌入模块。利用 asyncio.Semaphore 控制 QPS，实现带 Jitter 的指数退避重试策略拦截 429 限流报错。将 50 万字符文本平稳导入 Qdrant。

高可用导入 Batch Embedding Semaphore Exponential Backoff tenacity
40
Day

非结构化文本清洗、规范化与字段分流

实现 CleanTextPipeline 类，完成正则清洗（乱码/控制字符/多余空格）、段落分段、SHA-256 去重哈希等全链路文本预处理。

数据预处理 正则清洗 SHA-256 字段分流

Day

跨语言检索与 Embedding 表征漂移检测

使用多语言 Embedding 模型（multilingual-e5）实现中英跨语言语义对齐检索。量化分析不同领域文本的表征漂移（Representation Drift）现象。

跨语言 multilingual-e5 Representation Drift 语义对齐

综合实战：高吞吐知识检索引擎

将 Day 36-41 的所有模块拼装为完整的"AI 研究助手"知识底座：文件监听 → 清洗 → 批量向量化 → Qdrant 索引 → Pre-Filtering 检索，全链路 15ms SLA。

各模块解决的系统痛点

模块 无此模块时的系统痛点 引入后的解决方案
Day 36
向量化数学		 对距离度量选择盲目，反讽/转折文本召回结果完全偏离语义 定量理解余弦/L2 适用场景，在表征退化边界处设置人工兜底策略
Day 37
HNSW 索引		 暴力扫描 10万+ 向量，单次检索耗时 200ms+，Agent 决策卡死 HNSW 图索引 O(log N) 近邻查找，检索延迟降至 5ms 以内
Day 38
Pre-Filtering		 Post-Filter 后有效结果为零，多租户场景发生越权数据泄露 Payload 索引 + Pre-Filtering 在引擎层同步过滤，杜绝空结果和越权
Day 39
限流保护		 暴力并发调用 Embedding API，触发 429 封锁，全量导入任务中断 Semaphore + 指数退避 + Jitter，自适应降速自愈，零中断完成导入
Day 40
文本清洗		 HTML 标签/控制字符直接送入 Embedding 模型，向量几何方向被噪声扭曲 CleanTextPipeline 规范化 + SHA-256 去重，向量表征精度提升
Day 41
跨语言检索		 中文 Query 无法匹配英文文档，Agent 全球化部署检索召回率为零 多语言 Embedding 模型实现跨语种语义对齐，量化表征漂移边界
