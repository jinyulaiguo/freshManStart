Week 6：Embedding 与现代 RAG 技术体系总结

本周核心主线：知识如何表示（Embedding）→ 知识如何存储（Vector DB）→ 知识如何检索（Retrieval）→ 知识如何优化（Modern RAG）→ Agent 如何使用知识（Agentic RAG）

最核心的思想

现代 RAG 已经不是简单的 "Embedding → 向量检索 → LLM" 静态流程。而是通过 Embedding、Chunking、Hybrid Search、Reranker 和 Agent 推理 共同构建最适合 LLM 推理的 Context（上下文）。

传统 RAG 目标：
"找到最相似的文档 (Find the Most Similar Documents)"
现代 RAG 目标：
"构造最适合模型推理的上下文 (Construct the Best Context for Reasoning)"

Phase 01

第一阶段：知识表示 (Knowledge Representation)

问题：计算机如何理解一段文本？

在进入 RAG 之前，首要问题是计算机如何理解并计算一段自然语言文本。文本向量化的核心链路如下：

文本 (Text)
↓
Tokenizer (分词) → Token 序列
↓
Transformer Encoder (语义提取)
↓
Hidden States (每个 Token 的上下文表示)
↓
Pooling (池化压缩)
↓
Sentence Embedding (句向量)
↓
高维语义空间 (Semantic Space)

关键认知：Embedding 并不是一句话的"压缩版"。而是 一句话在高维语义空间中的坐标 (Coordinate)。之后的所有检索，本质上都是在比较这些坐标之间的位置关系。

核心概念定义

Embedding (嵌入向量)
文本映射后的固定维度向量，作用是将自然语言转换为数学高维空间中的坐标点。
Semantic Space (语义空间)
高维向量空间。相似的语义会聚集在相近的邻域。
示例: Car → Automobile → Vehicle 对应的向量在这个高维空间中距离极其邻近。
Pooling (池化)
将 Token 序列的多个隐藏层向量压缩凝聚成单个句向量。常见方法包括 Mean Pooling、CLS Pooling、Attention Pooling。
Cosine Similarity (余弦相似度)
计算并比较两个向量在高维空间中方向的偏角余弦。不关心向量长度（文本长度带来的长模长），只关心方向，是语义检索的首选。
L2 Distance (欧氏距离)
比较两个高维空间点之间的绝对位移直线距离，会同时受方向与向量模长（长度）的影响。
Representation Dilution (表征稀释)
如果一个文本块（Chunk）中混杂了多个主题，Embedding 只能被迫拟合表示这多个主题的平均中间方向。因此，Chunk 越杂乱，Embedding 表征就越不准确。
