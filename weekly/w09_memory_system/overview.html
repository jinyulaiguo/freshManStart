微引擎架构拆解 (Micro-Engines Analysis)

系统被拆解为五个核心“微引擎”，每个模块只承担单一职责，物理隔离并配备独立的单元测试。点击查看痛点与解决方案：

1. Memory Router (记忆决策路由)
前置过滤

决定用户 Query 的检索流向，阻断无关检索。

2. Buffer Memory Manager (短期压缩)
工作上下文

基于非阻塞并发后台任务自动将旧历史消息压缩为摘要段落。

3. Fact Extractor (原子事实提取)
增量沉淀

精准剥离用户对话中沉淀下来的结构化永久偏好与事实。

4. Memory Consolidator (时序一致消解)
记忆清道夫

执行记忆合并、去重以及基于遗忘曲线的时间权重衰减。

5. Persistence Store (物理持久适配层)
数据库接入

处理 Session 反序列化重构，支持系统在意外中断后断电恢复。

Memory Router

🚨 解决的痛点问题 (Pain Point)

如果无差别地对每次用户提问（如日常寒暄 “你好”）执行向量检索，将直接导致首字延迟（TTFT）大幅飙升、API 产生大量冗余扣费，并因为引入无关的噪声上下文而导致模型回答出现幻觉。

✅ 具体做了什么 (What it does)

利用轻量化 Prompt 分类机制拦截 Query，智能路由分流至：`MEM`（仅召回个人偏好）、`RAG`（检索外部知识库）或 `NONE`（直接会话）。
