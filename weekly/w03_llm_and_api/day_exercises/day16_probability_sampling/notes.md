# Day 16：概率采样控制与 Agent 确定性路由策略

本节重点关注如何精细化控制大语言模型（LLM）API 的采样参数（Temperature、Top-P、Top-K），以及如何根据 Agent 的不同业务场景（如确定性路由 vs 创意反思）定制经验模型。

---

## 📖 核心概念讲解

### 1. Softmax 函数：从 Logits 到概率分布
大模型的最后一个线性层输出的是一组未归一化的对数概率值，称为 **Logits**。Softmax 函数的作用是将这组数值转化为总和为 1 且各个元素在 $[0, 1]$ 之间的概率分布：
$$P(x_i) = \frac{e^{z_i}}{\sum_{j} e^{z_j}}$$
其中 $z_i$ 为词表中第 $i$ 个 Token 的 Logits。

### 2. Temperature（温度）：控制输出的确定性与创造性
温度参数 $T$ 在执行 Softmax 之前对 Logits 进行缩放：
$$P(x_i) = \frac{e^{z_i / T}}{\sum_{j} e^{z_j / T}}$$

*   **当 $T \to 0$ 时（极低温度）**：
    最大 Logit 对应的 Token 概率会被无限放大，趋近于 100%（即 Argmax 行为）。模型的输出变得极其确定且可重复。
*   **当 $T \to 1.0$ 时（默认温度）**：
    模型保持原始训练时的概率分布进行采样。
*   **当 $T > 1.0$ 时（高温）**：
    Logits 之间的差距被平滑和拉近，低概率的词被选中的概率显著上升，输出变得极其丰富甚至混乱（高熵值）。

### 3. Top-P（核采样 / Nucleus Sampling）与 Top-K 采样
为了在高温环境下防止模型“胡言乱语”，我们需要对采样空间进行截断：
*   **Top-K 采样**：每次只保留 Logits 最高的 $K$ 个 Token，直接将第 $K+1$ 名及以后的 Token 概率置零。
*   **Top-P 采样**：对候选 Token 按照概率降序排列并进行累加，当累加值首次达到阈值 $p$（例如 $0.9$）时，将排在后面的所有长尾低概 Token 概率归零并重新归一化。

> [!IMPORTANT]
> **采样参数联动效应**：通常建议只调节 `temperature` 或 `top_p` 之一。如果同时调大 `temperature`，可以通过调低 `top_p`（例如 $0.3 \sim 0.5$）强制掐断长尾噪声，在保持适度创造性的同时确保基本格式的稳定性。

## 💡 补充总结与问答

### 1. Agent 在不同场景下的参数经验配置

| 场景类型 | 典型任务 | 建议参数配置 | 选型意图与痛点防范 |
| :--- | :--- | :--- | :--- |
| **确定性路由** | 提取工具参数、JSON 数据反序列化、API 路由选择 | `temperature = 0.0` (或接近 0)<br>`top_p = 1.0` | **格式崩溃防线**：杜绝因随机采样引入的格式破损（如缺失闭合括号 `}` ），确保解析 100% 成功。 |
| **纠错与反思** | 对失败的 Tool 结果进行诊断、重试生成、代码 Bug 修复 | `temperature = 0.5 ~ 0.7`<br>`top_p = 0.9` | **发散思维**：适度调高温度，允许模型尝试非首选的替代逻辑，打破死循环。 |
| **创意生成** | 方案草拟、多样化 Agent 扮演、文案润色 | `temperature = 1.0 ~ 1.5`<br>`top_p = 0.9 ~ 1.0` | **多样性最大化**：鼓励使用同义词或非典型句式，以产生更具创意的多变文本。 |

### 2. 采样逻辑常见报错与防御设计
*   **API BadRequestError (400 报错)**：
    有些接口（如部分国产大模型适配层）在传入 `temperature = 0` 时会抛出越界异常，因为其后台分母不允许为 0。
    *   *防御代码实现*：在包装 API 时使用 `temp = max(1e-6, min(temperature, 2.0))` 进行安全拦截。
*   **Logits 指数溢出**：
    在自定义 Softmax 时，直接计算 `math.exp(logit)` 如果 Logit 过大（如 800+）会导致 `OverflowError`。
    *   *防御代码实现*：利用平移技巧，对 Logits 向量先减去其最大值：`adjusted = [l - max_logit for l in logits]`，再进行指数运算。
