WEEK 11 ROADMAP

Week 11: LangGraph 进阶与生产级架构

核心目标：掌握人在回路（HITL）交互与运行时中断、状态“时间旅行”回滚、子图隔离与多节点并发归约，具备编写分布式持久化 Checkpointer 的能力，最终交付可商业落地的带安全审批与故障自愈的 SQL 执行 Agent。

🎯 本周实战大项目架构全景 (Day 77)

本周我们将构建一个企业级带人工安全审批与故障自愈的 SQL 智能交互系统。系统整体业务流程如下：

1

Node 1: 自然语言转 SQL 思考（Text2SQL）

Agent 接收用户问题，结合 Schema 进行反思推演并生成候选 SQL 语句。

2

HITL 安全断点拦截与审计 Interrupt Barrier

图执行至高危节点（SQL 执行）前强制挂起冻结，将导出的 SQL Payload 呈报外部人工。安全人员可原位批准，或通过 update_state 重写错误的 SQL 参数。

3

Node 2: 数据库执行与安全沙箱防护 DB Sandbox

在 SQLite/PostgreSQL 沙箱中原子化执行校验后的 SQL，实时捕获语法错误、锁等待或超时异常。

4

时间旅行故障恢复与分叉自愈 Time Travel Fork

若数据库执行崩溃，触发故障降级机制。工程师可查询 Checkpoint 历史版本，定位到旧点快照，重置 State 注入纠偏 Context，从断点发起分叉继续执行。

5

自定义 Redis Checkpointer 分布式持久化 Redis Store

底座采用继承 BaseCheckpointSaver 的分布式 Redis 持久化组件，支持多租户高并发锁与字节流状态无损还原。

🗺️ 每日知识链路与痛点攻克

DAY 71
HITL 中断（interrupt）与断点控制
💥 痛点：LLM 在高风险操作（删除、支付、写库）中不可控，直接运行易引发灾难事故。
✨ 方案：拓扑断点 interrupt_before/after 与动态 NodeInterrupt/interrupt() 双重冻结图控制流。
DAY 72
工具调用人工审批与运行时拦截重写
💥 痛点：LLM 生成的工具入参拼写错误时，重新生成耗时费 Token 且结果依然不稳定。
✨ 方案：利用 update_state 在挂起状态下原位覆写工具参数，实现秒级人工纠偏。
DAY 73
状态“时间旅行”（Time Travel）与回滚重新触发
💥 痛点：Agent 运行 10 步后第 9 步偏离，过去只能全部废弃重跑，严重浪费算力。
✨ 方案：基于 Checkpoint 历史快照 ID（checkpoint_id）发起特定节点分叉二次 invoke 运行。
DAY 74
子图（Subgraph）状态隔离与并发子流程设计
💥 痛点：大型系统单图状态过于臃肿，不同业务模块之间 State 字段互相污染缠绕。
✨ 方案：构建独立编排的 Subgraph，通过 Input/Output Schema 物理隔离局部高频消息。
DAY 75
多线程并行节点（Parallel Nodes）与分支汇聚
💥 痛点：多源独立查询（如多引擎搜索）顺序串行执行耗时太长，并发合并易丢失数据。
✨ 方案：拓扑多路并发分支 + 汇聚节点 Annotated 列表安全 Reducer 顺序强一致合并。
DAY 76
持久化存储与 Redis/Postgres Checkpointer 扩展
💥 痛点：默认 MemorySaver 随服务重启而清空，无法支撑生产环境高可用多租户集群。
✨ 方案：深入 BaseCheckpointSaver 契约，封装带并发锁与二进制序列化的 RedisCheckpointer。
