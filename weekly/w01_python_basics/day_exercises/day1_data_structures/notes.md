## 1. 基础数据类型与数据容器概述

*   **基本数据类型**：
    *   `str` (字符串)：大模型输入输出的核心载体（Prompt、Generated Text）。
    *   `int` & `float` (整型与浮点型)：主要用于 Token 计数、LLM 耗时统计、温度值（Temperature）等超参数设定。
    *   `bool` (布尔值)：用于 Agent 决策判断（例如：`finish_reason == "stop"` 决定是否终止循环）。
*   **容器的变与不变（Mutability）**：
    *   **可变对象 (Mutable)**：`list`, `dict`, `set`。可以原地修改，需防范“深拷贝与浅拷贝”引发的隐式数据篡改。
    *   **不可变对象 (Immutable)**：`str`, `int`, `float`, `tuple`。一经创建无法修改，适合作为字典的键（Key）。

## 2. 四大容器常规操作大纲

### 列表 (List) ── 可变与有序
*   **创建与初始化**：字面量 `[]`，类型转换 `list(iterable)`。
*   **增删查改**：
    *   新增：`append(x)` (尾部追加), `extend(iterable)` (合并), `insert(index, x)` (指定位置插入)。
    *   删除：`pop([index])` (弹出并返回), `remove(x)` (删除第一个匹配值), `clear()` (清空)。
    *   修改与查找：`list[index]`，切片操作 `list[start:stop:step]`。
*   **常用操作**：
    *   排序：`sort(key=...)` (原地排序), `sorted(iterable, key=...)` (返回新列表)。
    *   推导式：`[x for x in iterable if condition]`。
*   **Agent 场景**：存储消息历史（Memory List）、保存可选的 Tool 列表。

### 元组 (Tuple) ── 不可变与安全防护
*   **创建**：字面量 `(x, y)`，单元素元组必须加逗号 `(x,)`（如果不加逗号如 `(1)`，Python 会误识别为普通的圆括号整型数）。
*   **核心特性**：
    *   **不可变性 (Immutable)**：一旦定义不能修改（不能增删改元素），安全防篡改。
    *   **相对不可变性**：元组的不可变指的是其包含的每个元素的引用（内存地址）不可变。如果元组内包含可变对象（如列表），可以修改该列表内部的数据，但不能将该列表整体替换。
    *   **性能优越**：占用内存比列表小，创建速度快，运行效率高。
*   **高频操作**：
    *   解包 (Unpacking)：`a, b = (1, 2)`。
    *   带星号解包：`first, *middle, last = (1, 2, 3, 4)`。
    *   作为字典的键 (Key)：因为元组是不可变的（可哈希的），可以用作字典的键，如坐标 `(x, y)` 或多维参数。
*   **Agent 场景**：在函数中返回多维数据（例如返回：`(token_count, cost_seconds)` 复合元组）；或者存储一些不可变的连接/静态配置信息。

### 集合 (Set) ── 去重与集合运算
*   **创建**：字面量 `{x, y}`，空集合必须使用 `set()`（因为 `{}` 表示空字典）。
*   **核心特性**：无序、元素不可重复、元素必须可哈希（Hashable）。
*   **高频操作**：
    *   添加/删除：`add(x)`，`remove(x)` (元素不存在会报错)，`discard(x)` (元素不存在不报错)。
    *   集合运算（在 Agent 依赖分析与工具管理中常用）：
        *   交集：`set_a & set_b` 或 `set_a.intersection(set_b)`
        *   并集：`set_a | set_b` 或 `set_a.union(set_b)`
        *   差集：`set_a - set_b` 或 `set_a.difference(set_b)`
*   **Agent 场景**：对大模型提取的关键词进行去重；计算工具库支持的工具与用户请求的重合度。

### 四大容器综合对比与使用场景

#### ① 基础特征对比
| 容器类型 | 语法示例 (Syntax) | 是否有序 (Ordered) | 是否可变 (Mutable) | 元素可重复 | 查找与访问效率 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **列表 (List)** | `['a', 'b', 'c']` | **是** | **是** | **是** | • 按索引访问：$O(1)$<br>• 按值查找：$O(n)$ |
| **元组 (Tuple)** | `('a', 'b', 'c')`<br>单元素：`('a',)` | **是** | **否** | **是** | • 按索引访问：$O(1)$<br>• 按值查找：$O(n)$ |
| **集合 (Set)** | `{'a', 'b', 'c'}`<br>空集合：`set()` | **否** | **是** | **否** | • 无法按索引访问<br>• 按值查找（判断在不在）：**$O(1)$** |
| **字典 (Dict)** | `{'key': 'value'}` | **是** (3.7+ 起)<br>*(按插入顺序)* | **是** | **键**唯一<br>**值**可重复 | • 无法按索引访问<br>• 按键 (Key) 读写：**$O(1)$** |

#### ② 具体应用场景与实例对照
| 容器类型 | 基础开发场景与示例 | Agent / 大模型开发场景与示例 | 为什么选择它？（核心动机） |
| :--- | :--- | :--- | :--- |
| **列表 (List)** | **购物车商品列表**：<br>按用户添加的先后顺序展示商品。<br>`['item1', 'item2']` | **历史聊天上下文 (Memory)**：<br>必须严格保证对话顺序（用户先说，AI 后说）。<br>`[{'role': 'user'}, {'role': 'assistant'}]` | 需要**按顺序**追加数据，且数据内容和长度随时会变。 |
| **元组 (Tuple)** | **数据库连接配置**：<br>防止配置信息在程序运行中被误修改。<br>`('127.0.0.1', 3306, 'root')` | **多维结果返回**：<br>大模型调用函数同时返回消耗的 Token 和耗时。<br>`token_count, duration = (1024, 1.25)` | 保证数据**只读安全**；或者用于函数一次性返回多个结果（解包）。 |
| **集合 (Set)** | **文章标签过滤**：<br>将用户输入的重复 Tag 进行过滤去重。<br>`{'python', 'basics'}` | **可用工具交集匹配**：<br>快速比对大模型抽取的关键词与工具库的交集。<br>`needed_tools & registered_tools` | 元素**不能重复**；或者需要做高效的**交集、并集、差集**数学运算。 |
| **字典 (Dict)** | **用户信息档案**：<br>存储包含多种属性的结构化对象。<br>`{'id': 1001, 'status': 'active'}` | **工具注册表 (Tool Registry)**：<br>通过工具名称瞬间找到并执行对应的 Python 函数。<br>`{'search': search_func, 'calc': calc_func}` | 需要**键值映射**，能够通过一个唯一标识（Key）瞬间取出数据。 |

## 3. 字典 (Dict) 深度操作 ── Agent 的数据基石

### get() 默认值机制
*   **痛点**：使用 `dict[key]` 访问不存在的键会直接导致 `KeyError` 崩溃。
*   **安全获取**：`dict.get(key, default=None)`。
*   **高级细节**：
    *   `dict.setdefault(key, default)`：如果键不存在，则插入并返回默认值；如果存在，直接返回已有的值。这在统计词频或构建多层结构时很有用。

### 字典推导式 (Dict Comprehension)
*   **语法**：`{key_expr: value_expr for item in iterable if condition}`。
*   **高频使用场景**：
    *   **提取/过滤子集**：从 LLM 的海量 Meta 信息中只过滤我们需要的数据。
    *   **键值对调**：`{v: k for k, v in original_dict.items()}`（例如将“工具名称->工具函数”转换为“工具函数->工具名称”）。
    *   **Key 格式化**：例如将 API 返回的驼峰命名 Key 统一转为蛇形命名（Snake Case）。

### KeyError 错误预防与处理
*   **LBYL (看好了再跳)**：在访问前通过 `if key in dict_data` 校验是否存在。
*   **EAFP (先做再说，不行再认错)**：使用 `try-except KeyError` 结构进行捕获，适合处理复杂的深度嵌套。
*   **开箱即用容器**：
    *   `collections.defaultdict`：为不存在的键自动生成默认值对象（如列表、嵌套字典等）。
    *   `collections.UserDict`：需要自定义字典底层解析行为时的子类化工具。

### 嵌套字典安全解析 (Agent Payload 场景)
*   **问题模型**：
    大模型返回 payload：`{"choices": [{"message": {"content": "..."}}]}`。
    如果我们用传统方式 `data["choices"][0]["message"]["content"]`：
    1. 若 `choices` 为空列表 $\rightarrow$ `IndexError`。
    2. 若 `message` 字段缺失 $\rightarrow$ `KeyError`。
    3. 若整个返回报错 `{"error": "..."}` 而无 `choices` $\rightarrow$ `KeyError`。
*   **优雅的解决方案**：
    *   手写基于点分割路径的安全解析递归/遍历函数（详见同目录下的 [practice.py](file:///Users/zhouyi/03.AI/03.freshManStart/weekly/w01_python_basics/day_exercises/day1_data_structures/practice.py)）。

## 4. 个人总结

### Union 的作用
* **类型注解**
    * 声明多种允许的类型：它向调用者和类型检查工具声明，data 参数既可以接受 dict（字典）类型，也可以接受 list（列表）类型。
    * 提升代码可读性：明确告知开发者此函数支持处理嵌套字典或嵌套列表结构。

### safe_get 安全设计理念与参数说明
* **功能意义**：防御性编程，避免提取深层嵌套字典或列表时由于键不存在 (`KeyError`)、索引越界 (`IndexError`) 或类型不匹配 (`TypeError`) 而导致程序崩溃。
* **参数说明**：
    * `data`：目标嵌套数据源。
    * `path`：寻找数据的路径（支持 `.` 分割的字符串或 `list` 形式的混合索引）。
    * `default`：当路径匹配失败时的备用默认值（默认返回 `None`）。
