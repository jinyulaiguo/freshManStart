微引擎与核心机制拆解 (Core Engines & Mechanisms)

LangGraph 框架内部由多个精密设计的子契约与微引擎协作构成。点击查看它们的痛点与解决方案：

1. Runnable Protocol (可运行契约)
接口契约

统一大模型组件输入与输出通道，支持标准化异步与流式生命周期。

2. StateGraph Compiler (状态图编译器)
编译引擎

将显式的图节点与有向边编译成具备复杂流转控制的执行网络。

3. State Reducer (状态归约引擎)
状态管理

防止节点覆盖更新导致的上下文丢失，为消息或数值字段提供增量合并逻辑。

4. Conditional Router (条件分支决策)
自主路由

基于状态上下文动态判断流转路径，实现 ReAct 循环中的工具分流。

5. Checkpoint Saver (快照持久化隔离)
会话安全

通过 `thread_id` 绝对隔离用户会话，为每次状态演进记录物理快照，支持宕机重续。

1. Runnable Protocol (可运行契约)

🚨 解决的痛点问题 (Pain Point)

在大模型应用开发中，各个底层组件（大模型、检索器、Prompt 模板）的方法调用千差万别，缺乏统一的方法生命周期契约。这导致无法做统一的异步化重试、缺乏统一的流式（stream）数据传输处理，且基于管道符 `|` 的旧 LCEL 链难以应对分支和环路。

✅ 具体做了什么 (What it does)

定义了标准的 Runnable 协议，统一声明了 `invoke`/`ainvoke`（单次调用）和 `stream`/`astream`（流式输出）方法矩阵。所有 LangGraph 节点本质上都是符合该协议的 Runnable 实体，从而保证了拓扑图的统一输入/输出校验。
