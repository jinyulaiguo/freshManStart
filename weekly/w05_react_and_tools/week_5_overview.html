核心模块痛点映射矩阵

组成模块 具体做了什么 解决了什么痛点问题
主控制环 & FSM 驱动异步 while 循环，调配 LLM 推理、工具分发和步数截断。 痛点 缺乏生命周期管控，发生死循环或无边界运行，导致资源无限消耗。
反射注册中心 (ToolRegistry) 利用 inspect 与 Pydantic 动态提取函数参数，自动输出规范的 JSON Schema。 痛点 手写 JSON Schema 繁琐易错；缺乏运行时类型拦截，导致传入无效参数破坏工具执行。
死循环检测器 (StuckDetector) 利用滑动窗口计算 Action 与参数的哈希值，检测重复调用频次。 痛点 模型因参数或逻辑打转，在循环中连续请求相同工具（入参一致），造成极高 Token 费账单。
并发调度层 并发协程运行，隔离故障抛错，确保部分成功工具正常返回。 痛点 串行执行导致严重延迟（TTFT 过高）；单个工具崩溃直接引起整个并发批次任务瓦解。
自愈反思模块 捕获运行时异常（ValueError、超时等），转化为格式化纠错 Observation 引导 LLM 自主订正。 痛点 网络波动或参数格式微瑕，使得 Agent 遇到细微异常就直接中断退出，健壮性极差。
